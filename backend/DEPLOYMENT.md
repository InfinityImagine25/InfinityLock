# Infinity Lock Admin Panel - Deployment Documentation

## Overview
This document provides instructions for deploying and accessing the Infinity Lock Admin Panel, including Super-Admin setup and authentication.

## Admin Panel URL
- **Login Page**: `/login`
- **Dashboard**: `/dashboard` (after authentication)

## Super-Admin Account

### Initial Credentials
A Super-Admin account is automatically created on first database initialization.

**Email**: `infinityimagine@outlook.com`  
**Role**: Super Admin (highest privilege level)  
**Status**: Active  

### Retrieving Initial Credentials
The initial password and TOTP secret are stored in:
```
/app/backend/initial_credentials.json
```

This file contains:
- `email`: Super-Admin email
- `initial_password`: Generated secure password (20 characters)
- `totp_secret`: Secret for Google Authenticator/Authy setup
- `provisioning_uri`: QR code URI for authenticator apps
- `admin_panel_url`: Login page URL

### TOTP Setup Instructions
1. Open Google Authenticator, Authy, or any TOTP-compatible app
2. Add a new account using manual entry
3. Enter the `totp_secret` from the credentials file
4. Account name: `InfinityLock Admin:infinityimagine@outlook.com`
5. The app will generate 6-digit codes that refresh every 30 seconds

### First Login Steps
1. Navigate to `/login`
2. Enter the Super-Admin email
3. Enter the initial password from credentials file
4. You'll be prompted for TOTP verification
5. Enter the 6-digit code from your authenticator app
6. **IMMEDIATELY CHANGE YOUR PASSWORD** after first login

## Security Requirements
- TOTP MFA is **mandatory** for Super-Admin accounts
- Initial password should be changed immediately
- Credentials file should be deleted or secured after setup
- All login attempts are logged in Security Logs

## Role-Based Access Control (RBAC)

### Roles
| Role | Permissions |
|------|-------------|
| Super Admin | Full access: all admin features + deployment controls + security logs |
| Admin | User management, analytics, feedback, system health, settings (read-only) |

### Super-Admin Exclusive Features
- Security Logs viewing
- Settings modification (Privacy Policy, Terms of Service)
- Admin user creation

## Seed Script

### Automatic Execution
The Super-Admin seed script runs automatically on first server startup if no Super-Admin exists in the database.

### Manual Execution
```bash
cd /app/backend
python seed_superadmin.py
```

### Idempotent Behavior
- The script checks if a Super-Admin already exists
- If exists: skips creation (no duplicates)
- If not exists: creates account and saves credentials

### Secure Deletion (Production Handoff)
```bash
cd /app/backend
python seed_superadmin.py --delete --confirm
```
This removes the Super-Admin account and credentials file.

## API Endpoints

### Authentication
- `POST /api/auth/login` - Login with email/password
- `POST /api/auth/verify-totp` - Verify TOTP code
- `GET /api/auth/me` - Get current user info
- `GET /api/auth/totp-setup` - Get TOTP QR code

### Admin Management (Super Admin only)
- `GET /api/admin/list` - List all admins
- `POST /api/admin/create` - Create new admin

### User Management
- `GET /api/users/list` - List app users
- `POST /api/users/action` - Suspend/deactivate/resume users

### Analytics
- `GET /api/analytics/dashboard` - Dashboard stats
- `GET /api/analytics/features` - Feature analytics
- `GET /api/analytics/system-health` - System health metrics

### Security Logs (Super Admin only)
- `GET /api/security-logs` - Audit trail

MONGO_URL=mongodb://localhost:27017

SUPER_ADMIN_EMAIL=infinityimagine@outlook.com

### Brute-Force Protection

#### Admin Panel
- 2 failures = 5-minute lockout + email alert
- 3 failures = 1-hour lockout + email alert

#### Super Admin
- 1 failure = immediate lockout + email alert + OTP re-verification required
